A typical program usually contains a lot of different objects. Usually, we need to store them right and do some actions. For example, we can sort them. For doing this, you can use arrays until you bump into a lack of functionality. Let's say, if you're not sure about the size of your array and it turns out to be longer than you expected, it'll be quite hard to extend it in the future. That's why you need to meet MutableMap.

Definition

So, what is MutableMap? In a nutshell, it's just a container for different objects. We also call it a collection. According to the official documentation, it holds pairs of keys and related objects.

The latter ones are simply values and we can use any kind of objects as keys and values. The mutable map type looks like MutableMap<K, V> where K is a key type and V is a value type. You can set these types.

Before we'll move on, there is one major assumption, that is, keys in MutableMap are Strings and integer numbers. Anything that is hashable and immutable can be a key, but we will explain it in further topics. For example, don't use arrays as keys.

Initialization

Let's create our first MutableMap:

val marathon = mutableMapOf(1 to "John Doe", 2 to "Marty McFly", 3 to "Marie Curie")

There are three pairs: people and their positions. Numbers are keys and strings are values.

The compiler will automatically define types of objects. It sounds pretty convenient.

However, if you define an empty map, you will have to specify the type explicitly:

val marathon = mutableMapOf<Int, String>()

Accessing elements

OK, we have a map with no elements. Nice, now we can put some athletes' names regarding their places taken in the competition:

marathon[1] = "John Doe"
marathon[2] = "Marty McFly"
marathon[3] = "Marie Curie"

Now our map contains the same pairs as the map we defined earlier.

Let's take a look at the content of our map: print(marathon) will output {1=John Doe, 2=Marty McFly, 3=Marie Curie}.

We can easily get a specific value from the map: print(marathon[1]) gives you John Doe. Oh, look: we have access to the elements just like in an array, but instead of indexes, we have keys. Also, their number is almost unlimited.

Removing elements

What else can we do to our MutableMap? We've had already created marathon and added some athletes, now we're going to remove a value by its key:

marathon.remove(0)

This does nothing because there is no such key in the map. In other words, the map remains unchanged.

Or we can remove the entry for the specified key only if it is mapped to the specified value:

marathon.remove(1, "John Doe...")

The piece of code above again won't delete anything, because we input the wrong value.

We can also clear our map with the help of this function:

marathon.clear()

Now it's empty.

Checking elements existence

We can move on and check if an entry exists via containsKey() and containsValue() functions to make sure that we've removed everything from the map:

print(marathon.containsValue("John Doe"))
print(marathon.containsKey(2))
// output: false, false

Iterating through elements

Great! let's fill marathon with new entries and see how we can iterate through it:

marathon[1] = "Pickle Rick"
marathon[2] = "Rick Sanchez"
for (entry in marathon) {
    println(entry.value)
}
/* output:
Pickle Rick
Rick Sanchez
 */

Great! Each entry consists of a key and its value, thus, we can access them both inside the cycle. What is more, we can even modify the cycle like this:

for ((key, value) in marathon) {
    println(value)
}

It will be absolutely the same as the previous example.

One more thing: be careful with the ordering of MutableMap, because it will iterate in the order in which the entries were put but not in natural ordering. If you want to keep your entries ordered in a collection then you probably need to pick out something that we will explore later.

Counting elements

There are some more useful properties and functions:

  1. size shows how many entries there are in the map;
  2. isEmpty() shows whether a map is empty or not.

Conclusion

Now you know how to keep and process your objects in a more convenient way through MutableMap in Kotlin. Don't forget that you can store any kind of hashable object as a key and whatever you want as its value. You've learned many functions of MutableMap and now you're ready to dive into tasks.
